🧠 O que é Organização de Interface?
Antes de adicionar cores, imagens ou tipografia, um designer precisa organizar a informação.
Uma interface organizada permite que o usuário compreenda rapidamente o conteúdo apresentado.
Interface desorganizada gera
dificuldade de leitura;
sensação de confusão;
aumento do tempo para encontrar informações;
erros de navegação.
Objetivo do designer
O principal objetivo do designer é reduzir o esforço cognitivo do usuário.

Uma boa interface é aquela que quase não precisa ser explicada.

📐 O que é um Grid?
Grid significa grade.
É um conjunto de linhas imaginárias que organiza todos os elementos da interface. Praticamente todos os sites modernos utilizam grids.
Sem Grid
Elementos ficam desalinhados.
Espaçamentos diferentes.
Aspecto amador.
Com Grid
Organização.
Consistência.
Facilidade para programar.
Responsividade.
📊 Tipos de Grid
Grid de Colunas
Mais utilizado para websites.

Exemplo:
12 colunas
Margens laterais
Gutters

É o padrão utilizado pelo Bootstrap.
Grid Modular
Divide a tela em linhas e colunas.

Muito utilizado em dashboards.
Baseline Grid
Utilizado para alinhar textos.

Melhora bastante a leitura.
12 Grid de 12 Colunas
É o padrão mais utilizado.
Porque pode ser dividido facilmente em 12, 6, 4, 3, 2 e 1. Isso facilita muito a criação de layouts responsivos.

↔️ Margens, Gutter, Alinhamento e Espaçamento
Margens
Margem é o espaço entre o conteúdo e a borda da tela.

Nunca encoste elementos na lateral.

Exemplo Desktop:
| margem | Conteúdo | margem |
Gutter
É o espaço entre as colunas.

Não devemos colocar elementos ocupando parcialmente um gutter.
Alinhamento
Alinhamento à esquerda: mais utilizado e facilita leitura.

Centralizado: ideal para títulos e pouco recomendado para textos longos.

Direita: pouco utilizado, normalmente apenas para números.

🧩 Princípios de Organização Visual
Princípio da Proximidade
Itens relacionados ficam próximos.

Exemplo:
Nome
E-mail
Telefone

Todos pertencem ao mesmo grupo.
Princípio do Alinhamento
Todos os elementos devem seguir uma linha imaginária.

Nunca posicione elementos "no olho".

Utilize sempre as ferramentas de alinhamento do Figma.

✅ Boas Práticas de UI
Sempre utilize Grid.
Nomeie Frames.
Agrupe elementos relacionados.
Mantenha espaçamentos consistentes.
Utilize poucos tamanhos de fonte.
Evite excesso de informações.
Priorize a leitura.
Organize as camadas.
Mantenha o arquivo limpo.
